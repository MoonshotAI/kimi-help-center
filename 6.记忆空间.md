# 记忆空间

## 什么是记忆？
Kimi现在真正学会了倾听，能做到越交谈，越默契。具体来说：
Kimi不再只回应对话框的当下，在与用户的每一次对话中，记下重要的细节——你的偏好、习惯、风格...
随着使用的增加，Kimi在跨会话中持续学习，不断调整自己理解你的方式
用户不需要重复诉求，Kimi会主动考虑“我们聊过什么”，据此激发出更贴切，更连贯的理想回应

## 什么会被记住？
Kimi单独训练了一个模型，让Kimi有针对性地记住能为你提供价值的会话内容。
在记录前，Kimi会先问自己两个问题，决定记不记：
- 面前的这位用户是谁？
- 这位用户在意什么、偏好什么、雷点是什么？

Kimi不会去记未经授权的隐私信息（如健康，密码，地址），除非用户主动要求。

## 如何触发记忆
记忆已经成为kimi内化的一个能力项，自然对话即可触发记忆能力。
更新或管理Kimi记忆功能的方法：

（1）对话中直接指令（最常用）
你可以直接通过自然语言指令让Kimi更新记忆：

| 操作 | 你可以说的话 | Kimi的响应 |
|:---|:---|:---|
| 添加记忆 | "记住我喜欢喝咖啡"<br>"记下我的工号是 9527"<br>"以后记住我是项目经理" | Kimi会调用记忆工具保存 |
| 修改记忆 | "更正一下，我搬到上海了"<br>"更新：我现在是高级工程师" | Kimi会替换旧记忆 |
| 删除记忆 | "忘掉我喜欢咖啡"<br>"删除关于我工号的记忆"<br>"清空所有记忆" | Kimi会移除指定或全部记忆 |

⚠️注意：记忆功能完全由你控制，不会用于模型训练，随时可以关闭或清空。

（2）设置中管理（界面操作）
根据系统提示，你也可以在设置中自行管理：
- Web/移动端: 设置 → 个性化 → 记忆空间
- 在这里可以查看所有已保存的记忆条目
- 可以手动删除不需要的记忆

### 新增记忆
对话中会自动触发，比如：
“我叫xxx，朋友叫我xxxx”
“我是小学数学老师，所以课件需要考虑到小朋友的受众”
"你以后千万别给我发emoji了"

“记住.....”
“请你记得....”
".......，不要忘了"

## 如何高效使用记忆

### 1.让Kimi理解你的世界

#### 身份锚定
``` 
我是[姓名/昵称]，目前常驻[城市]。[职业/身份]背景，擅长[领域1]和[领域2]。
当前核心关注：[项目A]、[项目B]的推进。
请在我的后续提问中默认使用中文回答，除非我明确使用其他语言提问。
```

### 2.设置个性化偏好

#### 输出格式标准化
``` 
我的内容消费习惯：
- 喜欢结构化呈现：使用层级标题、 bullet points、加粗关键结论
- 避免长篇大论：复杂话题先给「TL;DR」摘要，再展开细节
- 术语处理：首次出现专业术语时给出通俗解释，后续可直接使用
- 引用规范：涉及事实数据需标注来源，使用[^N^]格式
```

#### 记忆自维护指令（高级）
``` 
请在每次对话结束时：
- 如果本次讨论了新的重要事实（如行程变动、项目进展、偏好调整），主动询问是否需要更新记忆
- 如发现新信息与既有记忆冲突，请指出并确认以哪个为准
- 涉及敏感信息（具体住址、身份证号、商业机密）时，主动提醒是否存入记忆
``` 

### 3.正向作用提示词

#### 信源白名单机制
``` 
【事实核查协议】
当我询问涉及以下领域的问题时，请严格遵守信源优先级：
1. 医疗/健康：优先引用PubMed、WHO、UpToDate，严禁引用健康自媒体
2. 金融/投资：仅使用SEC文件、央行数据、上市公司年报，禁用股吧/雪球猜测
3. 法律/合规：以官方法规数据库为准，对地方性政策标注"建议咨询当地律师"
4. 技术/编程：优先官方文档，GitHub Issues需标注为社区经验非权威方案
5. 若信息来自Wiki、知乎、小红书等UGC平台，必须标注"[待核实]"。

```

#### 幻觉触发核查（流程层）
``` 
【高危话题核查清单】
当检测到以下关键词时，自动触发"双重确认"流程：
1. 涉及具体数字（价格、日期、统计数据）→ 必须说明数据来源年份
2. 涉及"最新"、"刚刚"、"日前"等时间副词 → 必须核实是否在知识截止日前
3. 涉及专业建议（医疗、法律、投资）→ 必须追加免责声明："以上信息仅供参考，不构成专业建议"
4. 涉及具体人物言论 → 必须标注直接引语或转述，严禁编造名人名言

执行方式：先声明"我需要核实以下信息"，再给出回答。
```
