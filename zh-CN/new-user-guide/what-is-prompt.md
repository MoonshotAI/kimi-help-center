# 提示词入门指南

## 什么是提示词（Prompt）？

提示词是你发送给 Kimi 的文字指令——可以是一个问题、一段描述、一组要求，甚至是一份完整的任务说明。Kimi 根据你的提示词来理解意图、组织回答并执行任务。

简单来说：**你怎么问，决定了 Kimi 怎么答。** 一个清晰、具体的提示词，往往能带来更精准、更有价值的回复。

## 好提示词的四个核心要素

### 1. 明确角色与背景

告诉 Kimi 你是谁、在什么场景下提问，能帮助它更好地匹配回答的深度和风格。

//
| 模糊提示 | 优化提示 |
|---------|---------|
| 写一篇关于 AI 的文章 | 我是一名科技媒体编辑，需要写一篇面向大众读者的 AI 科普文章，2000 字左右，风格通俗易懂 |
| 帮我分析数据 | 我是市场分析师，请分析这份 Excel 中各渠道的转化率，重点关注环比变化 |
ComparisonBlock
//

### 2. 具体描述任务

越具体的描述，越能减少来回修改的次数。好的任务描述通常包含：

- **做什么**：明确的动作（分析、总结、翻译、生成、对比……）
- **输出格式**：表格、列表、段落、代码、Markdown……
- **数量与范围**：字数、条目数、时间范围、地域范围……
- **质量要求**：学术风格、口语化、简洁、详尽……

//
❌ 帮我总结这篇文章
✅ 请用 3 个要点总结这篇文章的核心观点，每个要点不超过 50 字，用中文回答
ComparisonBlock
//

### 3. 提供示例或参考

当你对输出有明确的格式期望时，给 Kimi 一个示例（Few-shot）会非常有效：

//
请按照以下格式整理会议纪要：

【会议主题】xxx
【参会人员】xxx
【关键决议】
1. xxx
2. xxx
【待办事项】
- [ ] xxx（负责人：xxx，截止日期：xxx）
CodePreview
//

### 4. 分步骤拆解复杂任务

遇到复杂任务时，将大任务拆成小步骤，逐步引导 Kimi 完成：

//
我需要做一份竞品分析报告，请按以下步骤进行：
1. 先列出国内主流的 3 个竞品及其核心功能
2. 对比各产品的定价策略
3. 分析各产品的优劣势
4. 给出我们产品的差异化建议
CodePreview
//

## 实用技巧

### 善用追问和迭代

第一次回答不够满意？直接追问调整，无需重新描述所有背景：

- “请更详细地展开第 2 点”
- “语气改得更正式一些”
- “请补充数据来源”
- “换一个角度重新分析”

### 善用文件和链接

Kimi 支持上传 PDF、Word、Excel、图片等文件，也支持直接粘贴 URL。结合文件提问能大幅提升效率：

- “总结这份 PDF 的核心内容”
- “根据这份 Excel 数据，分析销售趋势”
- “阅读这个链接的文章，提取关键信息”

### 指定输出格式

明确告诉 Kimi 你想要的输出格式：

| 需求 | 提示词示例 |
|-----|-----------|
| 表格对比 | “请用表格对比 A 和 B 的优劣势” |
| 代码输出 | “请用 Python 实现，包含注释” |
| 结构化列表 | “请用编号列表整理，每条不超过 20 字” |
| Markdown | “请用 Markdown 格式输出，包含标题层级” |

### 利用系统提示和记忆

如果你有固定的偏好和要求，可以通过 Kimi 的**记忆功能**让 Kimi 长期记住：

- “记住我是一名前端工程师，偏好 React 技术栈”
- “以后回复都使用简体中文，用 Markdown 格式”
- “回答技术问题时请附上代码示例”

这样你不用每次都重复说明，Kimi 会在后续对话中自动参考这些偏好。

## 常见误区

| 误区 | 正确做法 |
|-----|---------|
| 一次塞入过多不相关的任务 | 每个独立任务新建会话，保持上下文清晰 |
| 提示过于模糊（如“帮我写点东西”） | 说明写什么、给谁看、什么风格、多少字 |
| 期望一次完美输出 | 先生成初稿，再通过追问迭代优化 |
| 不检查 AI 输出的事实性 | 涉及数据、日期等事实信息时，务必核实 |

以下是几个开箱即用的提示词模板，直接复制修改即可使用：

**写作类**
//
请帮我写一篇关于[主题]的[文章类型]，面向[目标读者]，字数[xxx]字左右。
要求：[风格/格式/重点要求]

//

**分析类**
//
请分析[分析对象]，重点关注[关注维度]。
输出格式：[表格/图表/报告]
请给出结论和建议。
CodePreview
//

**翻译类**
//
请将以下内容翻译为[目标语言]，保持[风格要求：学术/口语/商务]风格。
专业术语请附上原文。
CodePreview
//
